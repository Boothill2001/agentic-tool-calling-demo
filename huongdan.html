1 Tool Calling là gì? Tại sao không phải lúc nào cũng RAG?

RAG tuyệt vời cho câu hỏi về tài liệu (policy, docs, knowledge base). Nhưng khi user hỏi "Customer C001 mua hàng bao lâu rồi?" — đáp án nằm trong database, không phải document.

Tool Calling = LLM không trả lời trực tiếp, mà tạo ra một structured JSON mô tả: "Cần gọi tool gì, với tham số nào" → backend validate → execute → trả kết quả về LLM để format.

Giống như LLM là brain (ra quyết định), còn tools là hands (thực hiện).

Nguyên tắc cốt lõi: LLM KHÔNG BAO GIỜ trực tiếp query database. LLM chỉ tạo structured tool call → backend validate schema + permission → rồi mới execute. Đây là cách production system thật hoạt động.

2 5 Routes — LLM phân loại intent

1️⃣

SQL Database

Hỏi về customer, order, revenue, ticket → LLM chọn đúng query function + args → backend chạy parameterized SQL (không raw SQL từ LLM).

2️⃣

Calculator

Tính toán số học → AST-safe eval (không dùng eval()). Có thể chain với SQL: "Revenue tháng này vs tháng trước, thay đổi bao nhiêu %?"

3️⃣

RAG (Policy docs)

Hỏi về chính sách (refund, shipping, warranty) → Qdrant in-memory retrieval → LLM trả lời kèm citations [1][2].

4️⃣

Clarification

Thiếu tham số bắt buộc → LLM hỏi lại thay vì đoán. Ví dụ: "Check that customer" → "Customer nào? Cho em ID hoặc tên."

5️⃣ Human Approval (High-risk actions)

Cancel order, process refund = destructive action. LLM flag risk_level: high → UI hiện dialog xác nhận → chỉ execute khi user bấm Approve. Không bao giờ tự động cancel/refund.

5 Cài đặt từng bước (Windows)

Yêu cầu: Python 3.10+ và DeepSeek API key (lấy tại platform.deepseek.com — nạp $2 là dùng thoải mái).
  1. Clone repo 
    git clone https://github.com/Boothill2001/agentic-tool-calling-demo.git
cd agentic-tool-calling-demo
  2. Tạo môi trường ảo 
    python -m venv .venv
.venv\Scripts\activate
  3. Cài thư viện 
    pip install -r requirements.txt
  4. Cấu hình API key 
    copy .env.example .env
# Mở .env bằng Notepad, thay your-api-key-here bằng DeepSeek key thật
  5. Seed database — tạo SQLite DB với dữ liệu mẫu (~110 rows). 
    python seed_db.py
# Thấy "Database seeded: data/enterprise.db" là OK
  6. Chạy 
    streamlit run app.py
# Trình duyệt tự mở http://localhost:8501

9 Câu hỏi phỏng vấn thường gặp

Tại sao LLM không trực tiếp viết SQL?

Bảo mật: LLM có thể bị prompt injection → sinh SQL nguy hiểm (DROP TABLE, data leak). Thay vào đó, LLM chỉ chọn tool_name + args từ registry đã define sẵn, backend build parameterized query. Giống như principle of least privilege — LLM chỉ có quyền gọi function, không có quyền viết code.

Structured tool calling khác function calling của OpenAI thế nào?

Concept giống nhau: LLM output structured JSON thay vì free text. OpenAI function calling là API-level feature (model đã fine-tune cho JSON output). Project này implement application-level: custom prompt + Pydantic validation + permission layer — chạy được với bất kỳ LLM nào (DeepSeek, Claude, GPT), không phụ thuộc provider-specific feature.

Human approval trong production thật implement thế nào?

Demo dùng Streamlit dialog. Production thật: async workflow — tool call tạo approval request trong queue (Slack notification, email, dashboard) → authorized person approve/reject → system resume execution. Thêm timeout, escalation, audit log. Pattern giống Temporal/Step Functions.

Execution trace dùng để làm gì?

3 mục đích: (1) Debug — khi kết quả sai, trace cho biết sai ở bước nào (router classify sai? validator reject? tool error?). (2) Audit — compliance cần biết AI quyết định gì, dựa trên logic nào. (3) Improve — phân tích pattern từ traces để fine-tune router prompt hoặc thêm tools.

Sao không dùng LangChain Agent?

LangChain Agent là autonomous loop — LLM tự quyết tool nào, chạy, đọc kết quả, lại chọn tool tiếp. Khó kiểm soát, khó debug, unpredictable cost. Project này dùng single-turn structured output: LLM quyết 1 lần, backend validate rồi execute. Deterministic hơn, auditabe, production-safe hơn.

Scale lên production cần thêm gì?

Thay SQLite bằng PostgreSQL; thêm connection pool; async tool execution; rate limiting per user; caching (same query → same route → cache result); monitoring (latency per route, error rate, LLM cost tracking); A/B test router prompts; fallback khi LLM timeout.

10 Lỗi thường gặp & cách sửa

LỗiNguyên nhân & cách sửa

LLM trả JSON sai format DeepSeek thỉnh thoảng thêm markdown fence. Code đã handle: strip ```json...``` wrapper trước parse
